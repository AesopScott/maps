Completed VA-045: Shared memory for security agents

Claude's research output:
- Architecture: use a memory-contract pattern where agents contribute typed observations to an immutable append-only log rather than mutating shared state.
- Memory layers: operational layer for fast, low-sensitivity coordination; evidence layer for immutable forensic references; resolution layer for hypotheses, causal chains, corrections, and rejected leads.
- Metadata envelope: every observation carries classification, confidence, source agent, timestamp, provenance, incident role, visibility rules, and mutation policy.
- Guardrails: share patterns/counts by default, not raw sensitive events; redact PII before operational sharing; gate evidence by clearance; bind reads/writes to incident role; preserve contradictions instead of overwriting; make evidence append-only; require reasoned corrections in the resolution layer.
- Eval/proof: simulate lateral movement investigations with conflicting agent observations, measure leakage prevention, verify bad-fact preservation in postmortems, and test forensic reconstruction against tampering.
- Product recommendation from Claude: treat as an enabling component, not a standalone product now. Validate inside a broader SOC or incident platform first.

Vik's architecture-owner interpretation:
- Shared memory for security agents is a control-plane requirement, not just a convenience. Without a strict memory contract, multi-agent SOC work will either leak sensitive data, collapse into stale shared assumptions, or destroy the reasoning trail needed for incident review.
- The durable design should separate facts, evidence references, hypotheses, decisions, rejected hypotheses, and operator annotations. Agents should be allowed to add evidence and interpretations, but not silently erase prior conclusions.
- The key product primitive is a Provenance-Preserving SOC Memory Ledger: append-only where it matters, editable only with explicit correction records, filtered by clearance, and replayable for post-incident audit.
- The system must treat bad facts as first-class historical artifacts. A refuted hypothesis is not trash; it is part of the investigation's cognitive path and useful for postmortem, eval, and analyst training.
