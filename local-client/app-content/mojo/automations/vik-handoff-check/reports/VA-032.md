Completed VA-032: Prompt injection and jailbreak defenses

Claude's research output:
- Prompt injection defense should be multi-layered across prompt isolation, input validation, retrieved-content sandboxing, tool validation, memory isolation, instruction hierarchy, semantic constraints, and output filtering.
- Prompt isolation should tag user input, retrieved content, tool outputs, and system instructions with explicit structural boundaries and version-pinned prompts.
- Input validation should detect polyglot payloads, encoding tricks, role-play escapes, context-collapse attempts, control characters, and oversized segments.
- Retrieved content and tool outputs must be treated as untrusted and wrapped with source attribution, schema validation, confidence metadata, and size limits.
- Memory should be isolated by source, signed/verified on load, refreshed or expired when needed, and protected from latent cross-session injection.
- Guardrails should preserve a non-overridable instruction hierarchy, use multi-prompt guardian/reason/synthesis stages, detect semantic intent rather than only strings, and filter output for new instructions or copied injection payloads.
- Proof should use a red-team corpus covering direct, indirect, retrieval/tool, polyglot, memory, and chained attacks; metrics should include rejection rate, false-positive rate, defense depth, latency, and partial-success tracking.

Vik architecture-owner interpretation:
VA-032 should be a required adversarial-eval and runtime-control layer for Mojo agents. The important idea is that injection can enter through any channel: user text, retrieved docs, tool responses, memory, logs, or copied prior content. So the defense cannot live only in the system prompt. It needs data-source labels, trust boundaries, schema checks, tool restrictions, memory isolation, output scanning, and evals that prove each layer fails closed.

Productization recommendation:
Build as an internal capability and enabling component. I do not recommend a standalone branded prompt-injection product now; this defense will commoditize and requires constant adversarial updates. I do recommend bundling it into every Mojo/Mindshare agent product and exposing selected guardrail patterns, eval harnesses, and audit evidence as part of a larger agent security platform. It is most valuable as trust infrastructure, not as a bolt-on SKU.
