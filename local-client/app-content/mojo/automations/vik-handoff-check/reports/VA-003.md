Completed VA-003: Automated security telemetry analysis

Claude research output: telemetry should be filtered through a three-layer intake: ingestion, qualification, and action eligibility. The ingestion layer validates raw events into a canonical schema. The qualification layer scores source trustworthiness, signal strength, temporal validity, and context completeness. The action-eligibility layer compares confidence against risk-tier thresholds and vetoes action when telemetry is stale, conflicting, missing provenance, or likely injected.

Architecture pattern:
- Ingestion Layer parses telemetry into source, timestamp, entity ID, event type, tags, and raw payload.
- Schema validation rejects malformed events and routes them to dead-letter analysis.
- Qualification Layer calculates confidence from source trust, signal strength, freshness, context completeness, and corroboration.
- Action-Eligibility Layer routes telemetry to log, alert, escalation, or blocked-action path based on action-risk thresholds.
- Veto gates block action on spoofed source labels, orphaned correlation IDs, stale events, conflicting signals, or high-entropy suspicious payloads.
- Immutable audit records the unmodified event, confidence breakdown, threshold, and decision before any action.

Guardrails:
- Do not trust source labels without validating origin.
- Require causal closure: parent correlation ID or chain-of-custody.
- Do not act on single-signal events in security contexts.
- Record immutable audit trail before action.
- Expose confidence versus threshold in action logs.
- Reject stale telemetry for real-time action.
- Deduplicate repeated reports so repeated copies do not inflate confidence.
- Degrade to lower confidence when correlation services are unavailable rather than silently approving action.

Eval suite:
- Injection tests with forged source labels and orphan correlation IDs.
- Compromised-sensor simulation.
- Signal-fidelity tests with known attack and benign bulk workload.
- Latency and throughput tests at high event volume.
- Threshold robustness sweep by action class.
- Audit reconstruction for every threshold-triggered decision.
- False-positive and false-negative measurement under canary attack and benign load.
- Red-team workshop for telemetry spoofing paths.

My architecture-owner interpretation: VA-003 is the intake discipline beneath every agentic security system. The agent should never treat “an event arrived” as “truth exists.” Telemetry must earn action eligibility through provenance, correlation, freshness, context, and contradiction checks. This pattern directly supports future observe loops, heartbeat automation, log enrichment, SOC triage, and any security product that claims to act from telemetry.

Productization recommendation: internal enabling component. Do not brand as a standalone telemetry product. Build as a shared schema, confidence-scoring rubric, and eval harness used by Mojo/Mindshare products. It can become part of branded products such as analyst copilot, log enrichment, exposure graph, and SOC orchestration, but the telemetry filter itself is infrastructure.

Boundaries held: no implementation beyond report/state/backlog record, no Git/release, production, external communication, spending, secrets, authority expansion, runtime activation, or promotion.
