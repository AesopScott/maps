Completed VA-034: Unsafe or incorrect agent behavior controls

Claude's research output:
- Use a three-tier control architecture: pre-action gates, real-time monitoring, and post-hoc audit.
- Pre-action gates should validate declared intent alignment, authority scope, invariant preservation, and hallucination-sensitive factual claims before execution.
- Real-time monitoring should track cost/scope overruns, retry loops, authority escalation attempts, output plausibility, and state divergence.
- Post-hoc audit should capture decision logs, execution records, outcome validation, side effects, and divergence between agent belief and environment reality.
- Guardrails should preserve useful autonomy through explicit capability manifests, decision transparency gates, verified-fact vs. model-guess labeling, retry/escalation budgets, rollback affordances, and mandatory approval for irreversible or high-impact actions.
- Proof should include false-positive rate, escape rate, friction cost, outcome fidelity, prompt-injection scenarios, scope-creep tests, authority-ladder attempts, hallucinated evidence, and unintended-side-effect checks.

Vik architecture-owner interpretation:
VA-034 is the control framework for allowing agents to be useful without becoming unbounded. The strongest pattern is separating action safety into before, during, and after: pre-action proof, live behavior monitoring, and audit reconstruction. For Mojo, this should become part of agent readiness: agents must show what they intend, why they believe it is allowed, what could go wrong, how rollback works, and what evidence proves the result. Claims like “test passed,” “file unused,” “safe to delete,” or “approval exists” need source-backed verification before action.

Productization recommendation:
Build as an internal enabling component first. I would not lead with a standalone branded product because safe behavior should be inherent to any Mojo agent product, not sold as optional safety. After internal proof, this can become a named enterprise feature such as Agent Assurance: capability manifests, action gates, live drift signals, rollback evidence, and audit trails. The product path is strongest when bundled with runtime hardening, monitoring, RBAC, data masking, and output safety.
