Research output: log enrichment becomes unsafe when logs are treated as memory without provenance. Raw logs are event evidence. Memory is interpreted state. Those must stay separate. Normalization can corrupt meaning by collapsing uncertain, partial, duplicated, or context-bound events into confident durable facts.

Architecture pattern:

`Raw Log -> Parse -> Normalize -> Classify -> Confidence Score -> Memory Candidate -> Review/Gate -> Durable Memory`

Recommended normalized event schema:

```text

session_id

operation

object_ref
event_class
severity
status
raw_excerpt_ref
normalization_version
parser_version
confidence
ambiguity_flags
memory_candidate
memory_target
review_required
retention_class
```

Event classes:

- observation
- decision
- approval
- denial
- blocker
- handoff
- action_taken
- state_change
- risk_signal
- incident
- noise

Meaning preservation rules:

- Never upgrade uncertainty.
- Never remove actor and authority context.
- Never merge conflicting events into one clean summary.
- Never treat absence as denial.
- Never convert runtime output into role memory without gate.
- Never overwrite canonical source truth.
