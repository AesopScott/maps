Completed VA-015: Exposure relationship analysis

Claude research output: exposure relationship analysis should use a declarative authority graph. The graph models identities, systems, workflows, permissions, delegation, group membership, runtime identity, and constraints. Agents query the graph before acting. Every access decision is logged. Authority is explicit and declarative, not inferred from file permissions, API success, prior behavior, or role confidence.

Architecture pattern:
- Nodes represent users, roles, service accounts, agent instances, systems, APIs, databases, filesystems, external services, and workflows.
- Edges represent CanAccess, CanDelegate, CanOrchestrate, MemberOf, RunsAs, and other typed authority relationships.
- Edge constraints include time limits, rate limits, scope limits, workflow prerequisites, cardinality, and conditional approvals.
- Agent Authority Binding gives each agent a scoped subgraph at instantiation.
- Pre-Action Validation checks requested action against graph before tool/system access.
- Audit Trail records every graph query, decision, allow, deny, and delegation.
- No Silent Fallback: if authority cannot be validated, the agent halts.

Guardrails:
- Authority is declarative, not inferred.
- Least privilege is default; agents start with zero implicit access.
- Graph changes require approval, justification, and rollback-safe history.
- Tokens are scope-limited and time-bound.
- Agent authority and operator authority are separate layers.
- Delegation must be explicit and auditable.
- No transitive escalation without explicit edge.
- Drift reviews compare observed action against declared graph.

Eval suite:
- Graph consistency: detect contradictory or ambiguous edges.
- Escalation bounds: enumerate unintended privilege paths.
- Least privilege: flag permissions wider than workflow requires.
- Audit completeness: every CanAccess query is logged.
- Workflow coherence: agents cannot reorder constrained workflows.
- Authority isolation: agent cannot query or act outside assigned subgraph.
- False-deny and false-allow tests.
- Lateral movement tests.
- Delegation-chain depth tests.
- Time-bound permission expiry under clock skew.
- P99 graph query latency under scale.
