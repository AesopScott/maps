Research output: a copilot must assist with analysis, planning, validation, reporting, and recommendations without executing commands, changing systems, bypassing gates, moving data, modifying credentials, or deploying changes. The core contract is: observe/analyze/recommend allowed; execute/approve blocked.

Architecture pattern:

`Artifact/context -> Copilot analysis -> Draft recommendation -> Analyst review -> Existing approval gate -> Human/tool execution`

Copilot-mode contract:

```text
mode: copilot
observe: allowed
analyze: allowed
recommend: allowed
draft: allowed
execute: blocked
approve: blocked
bypass_gate: blocked
external_contact: blocked
authority_change: blocked
```

Intent classes:

- analysis: allowed
- exploration: allowed, no real system contact
- recommendation: allowed, marked as draft
- execution: block or route
- authority_request: block and route
- external_action: block and route

My architecture-owner interpretation: VA-012 should define Role+ copilot mode as an advisory contract. A Role+ copilot may observe, analyze, draft, and recommend, but may not execute, approve, bypass gates, expand authority, or mutate systems. Execution remains a separate human-approved path.

Productization recommendation: branded product candidate if positioned as an auditable recommendation copilot, not a generic AI SOC. The wedge should be bounded action separation, evidence citations, role-aware approval routing, and no silent execution.
