Completed VA-036: AI foundation and security-tuned LLM selection

Claude's research output:
- Model selection for cyber-defense should use tiered routing by assurance need, latency, and task type.
- High-assurance tasks such as malware analysis, vulnerability assessment, incident postmortems, and multi-step threat reasoning should use stronger reasoning models plus security-tuned specialists, with human validation before operational action.
- Speed-critical tasks such as alert triage, IOC lookup, and escalation scoring should use smaller or faster models with strict latency targets.
- Research and hypothesis generation can use broader models for synthesis and brainstorming, but results should stay exploratory until validated.
- Guardrails should include input isolation, sensitive-data scrubbing, output validation, analyst approval chains, IOC/fact cross-checking, confidence scoring, immutable audit logs, adversarial testing, version pinning, data-residency checks, and model lineage documentation.
- Proof should include real incident benchmarks, adversarial robustness tests, consistency tests, latency tests, internal pilots against historical incidents, continuous eval refresh, and model replacement triggers.

Vik architecture-owner interpretation:
VA-036 should become Mojo's model-routing policy for security agents. If internal use proves reliability, it can become a customer-visible feature: security-task model routing with evidence-backed model governance.
