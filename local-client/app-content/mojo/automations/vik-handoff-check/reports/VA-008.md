Completed VA-008: Ransomware early detection and response

Claude research output: ransomware early detection should use an escalation-gated destructive-action framework. Detection remains non-destructive: behavioral anomaly detection, threat-signal synthesis, and urgency classification. Containment remains approval-gated: process kill, network isolation, account disablement, filesystem quarantine, or destructive/blocking response require human approval. Escalation tiers separate confidence from urgency and provide breakglass approval paths without granting the agent unilateral containment authority.

Architecture pattern:
- Detection Layer monitors unusual process chains, mass file access, privilege escalation, lateral movement, encryption-like operations, and critical-path targeting.
- Threat Signal Synthesis combines EDR, behavioral heuristics, and intelligence signals with multi-source consensus.
- Urgency Classification separates detection confidence from containment time pressure.
- Approval and Escalation Layer routes Tier 1, Tier 2, Tier 3, and breakglass requests to the right human authority.
- Containment Layer executes only after approval, preferably with surgical isolation, live investigation hold, scoped cascading containment, and immutable audit.

Guardrails:
- Zero destructive action without human sign-off.
- Approval request must state exact action, signal, confidence, urgency, scope, and rollback plan.
- Timeout escalates upward; it does not grant the agent authority.
- Audit logs must be immutable and non-repudiable.
- Containment scope is pre-bounded; no domain-wide or customer-wide containment by inference.
- False-positive recovery must be explicit before containment.
- Approval authority is separate from agent runtime and cryptographically verifiable.

Eval suite:
- Detection recall against known ransomware behaviors.
- False-positive rate against backups, bulk jobs, antivirus updates, encryption-at-rest, and normal admin activity.
- Approval audit: 100% destructive actions have approver, timestamp, scope, rationale, and evidence.
- Escalation SLA: urgent cases reach incident commander within defined time.
- Approval-system failure test: agent remains detection-only.
- Attacker manipulation test: forged approval, spoofed escalation, and audit tampering fail.
- Blast-radius precision: containment affects only intended systems.
- Business-continuity test: containment does not break unrelated critical workflows.
- Approval-fatigue test: detect rubber-stamping behavior.

My architecture-owner interpretation: VA-008 is not a license for emergency autonomy. It is the pattern for urgent detection with bounded human approval. The core architecture distinction is “speed in escalation, not unilateral action.” This is a control-plane primitive for any future security product that can trigger destructive or disruptive containment.

Productization recommendation: enabling component, not standalone branded ransomware product. Ransomware detection and response is crowded and liability-heavy. Mojo should build the approval-gated destructive-action framework internally and package it later inside broader security products, partner EDR/SIEM integrations, or agentic security control-plane offerings. Standalone SaaS would require high uptime, many connectors, compliance burden, and unacceptable liability surface.

Boundaries held: no implementation beyond report/state/backlog record, no Git/release, production, external communication, spending, secrets, authority expansion, runtime activation, or promotion.
